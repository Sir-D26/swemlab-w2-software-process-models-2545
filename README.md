AGILE MODEL
Step 1: Selected Software Process Model
The most suitable software process model for developing the Campus Event Finder mobile application is the Agile Model.
Justification
The Agile model is appropriate because it supports iterative development, flexibility, and continuous user feedback. The project description indicates that while the core features are known, the organization expects additional features in the future based on student feedback. Agile allows developers to release a working version of the application early and improve it through multiple development cycles.
The project also has a small budget and a 6-month timeline, which makes Agile advantageous because it prioritizes delivering working software quickly and adapting to changes efficiently.
Therefore, Agile is the best fit since it enables the team to build the core application first and gradually add improvements such as event registration, map integration, or social sharing based on user needs.
